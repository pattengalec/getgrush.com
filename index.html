What grush is

Most growing operations run on memory, notebooks, and spreadsheets. grush replaces that with one field-first tool: big buttons, no jargon, works with dirt on your hands or gloves on. Staff log the work; the record builds itself; the operator sees everything.

It isn't built for one crop or one place. The same software runs an outdoor teaching garden and an indoor container mushroom farm today — because a growing operation is a growing operation, whatever it grows.

Built to scale

grush is designed to run the same from the smallest grow to the largest — modular, like the container farm it manages.

It isn't rocket science — but it's built so it could be. The same tap‑not‑type tool that runs a backyard bed would run a grow on the moon: gloved hands, sealed suit, no room for error. That's what simple is for.

Why I built this

My path doesn't read like a straight line — and that's the point. I've been a Marine linguist, a 911 operator, and an armed power‑plant guard; a nurse through cardiac ICU, emergency, trauma, and prison units; and an electrician, a builder, and a project coordinator on the power‑distribution buildings that AI data centers and utility companies across the country depend on. Every one of those jobs came down to the same thing: the person on the ground had to get it right the first time, under real pressure, with real consequences.

Being a linguist taught me the lesson that shaped this: all the skill in the world doesn't matter if the team can't communicate. The best plan fails when the person who needs it can't read it, or the person who has the answer can't get it where it needs to go. grush is built around that — simple enough to be approachable to anyone, in whatever language they work in, so the knowledge actually moves.

Now, finishing a degree in environmental science, I'm building the tool I wish those systems had: one that turns the daily work into a record you can actually trust. I taught myself the whole stack — the software and the hardware — the same way I learned everything else: because the job needed doing.

grush is that idea applied to growing things. Every part of it is built for the person with dirt on their hands and the person running the operation alike.

I'm a 100% disabled Marine veteran and a father of four — one of them an active‑duty Marine. I build things meant to be run by someone other than the person who built them.

The working model

Fun Guy Fungi is the first grush business built on the platform: a single 40ft high‑cube container (~320 sq ft), single-operator, dual-door one-way workflow, targeting ~300+ lbs/week at capacity — and modular-expandable by adding containers, exactly as grush is built to scale.

The model has three parts that feed each other. Operate flagship grows like Fun Guy Fungi — real revenue and living proof the system works. Deploy the grush platform to other growing operations, each branded as its own. Scale every grow modularly, from a single container toward mapped acreage.

On the roadmap: large‑scale mapped operations (ArcGIS), and a restoration‑products line that turns spent growing material into soil amendments for wildfire‑scarred land — circular by design.
