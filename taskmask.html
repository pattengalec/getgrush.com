How It Works

Drag. Theme.
Work. Dissolve.

TaskMask overlays contextual visual identity onto any open application window. No plugins, no reconfiguration, no workflow disruption. The system reads the active app, generates a theme, and removes itself when you're done.

1
Drag Task

Pull a task card from your list and drop it onto any open application window — VS Code, a browser tab, a document editor.

2
AI Generates Theme

The AI abstraction layer reads the task context and target application, then generates a full visual theme: colors, fonts, ambient tone, intensity.

3
Overlay Applies

A render adapter applies the theme to the window surface — border glow, color grading, overlay UI — without modifying the underlying application.

4
Auto-Dissolves

When the session ends, the task is marked complete, or the window closes, the overlay dissolves cleanly. No residue, no cleanup, no config to undo.

Architecture

AI Abstraction Layer.
Render Adapters.

TaskMask is built around two patented core systems: an AI abstraction layer that generates theme specifications, and a render adapter system that applies them to target environments without modification.

Core System 1
AI Abstraction Layer (130–135)
Receives task metadata and target application context. Passes inputs to an LLM inference engine that generates a 20-field theme specification. The abstraction layer is provider-agnostic — swappable between OpenAI, Anthropic, or local inference without changing downstream render logic.
LLM Inference Provider-Agnostic 20-Field Schema Task Context
Core System 2
Render Adapter System (140–146)
Translates theme specifications into platform-specific rendering instructions. Adapters for Windows 11 DWM, macOS Core Animation, web environments, and Electron are implemented independently, allowing any adapter to be added without modifying the core theme engine.
Windows 11 DWM macOS Core Animation Web / Browser Electron
Interaction System
Drag-and-Drop Assignment (Steps 201–208)
User initiates a drag gesture from a task card. The system detects the target window under cursor, resolves the application identity, invokes the AI layer with full context, receives the generated theme, and applies it through the appropriate render adapter — all within a single interaction.
Window Detection App Identity Resolver Context Assembly One-Gesture UX
Lifecycle System
Session Lifecycle (Steps 301–306)
Themes are bound to sessions, not windows. Each session tracks task assignment, active application, and elapsed time. Dissolution triggers fire automatically on task completion, window close, or manual override — leaving no persistent state behind in the target application.
Session Binding Auto-Dissolve Zero Residue State Isolation

Theme Schema

20-Field Specification.
AI-Generated Per Session.

Every TaskMask theme is a structured 20-field specification generated fresh by the AI abstraction layer for each task-window pair. No pre-built themes, no user configuration.

primary_color
Dominant hue — derived from task category and urgency
secondary_color
Supporting accent, harmonically related to primary
background_tone
Base surface tint for the overlay environment
border_glow
Window edge illumination color and intensity
font_family
Typeface selection appropriate to task cognitive mode
font_weight
Weight tuned to reading vs. creation vs. review context
ambient_mode
Focus / Flow / Review / Sprint — drives overall tone
intensity
0–100 scale controlling overlay visual saturation
animation_style
Static / Subtle / Pulse — matched to task urgency
dissolve_trigger
Condition set for automatic theme removal
+ 10 additional fields covering overlay opacity, corner radius, taskbar integration, notification suppression, and session metadata.

Intellectual Property

Pro Se Provisional.
12-Month Priority Clock Running.

TaskMask Provisional Patent Application

Filed pro se as micro entity at USPTO Patent Center. Reference: TaskMask-PROV-001. The provisional establishes a 12-month priority date covering the core systems — AI abstraction layer, render adapter architecture, drag-and-drop assignment, session lifecycle, and the 20-field theme schema.

1.System for generating context-aware visual themes by associating a user task with a target application window via drag-and-drop gesture
2.System wherein a render adapter applies the generated theme to the window surface without modifying the underlying application
3.Method for assigning a task to an application window, generating a theme, applying via adapter, and dissolving the theme upon session completion
4.Method wherein the theme specification includes at least 10 visual and behavioral parameters generated by a large language model
5.AI abstraction layer that receives task metadata and target application context and outputs a structured theme specification via LLM inference
6.Render adapter system that translates theme specifications into platform-specific rendering instructions across multiple operating environments
